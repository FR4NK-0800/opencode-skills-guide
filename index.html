OpenCode Skills

Guia completa sobre como crear, instalar y usar Skills en OpenCode para potenciar tu agente de IA.

Incluye resumenes: El Forastero Misterioso (Mark Twain) y Las Aventuras de Pinocho (Carlo Collodi)

Que son los Skills?

Los Skills son conjuntos de instrucciones reutilizables que el agente de IA descubre y ejecuta automaticamente cuando detecta que son relevantes para la tarea. Piensa en ellos como "recetas especializadas" que le ensenan a OpenCode como manejar tareas especificas.

Automáticos

El agente carga el skill cuando la descripcion coincide con lo que el usuario pide. No necesitas invocar nada manualmente.

Reutilizables

Crea una vez, usa en cualquier proyecto. Los skills globales estan disponibles en todas partes.

Estandar Abierto

Creados por Anthropic, compatibles con Claude Code, OpenCode, Cursor, Copilot y 40+ herramientas mas.

Estructura de un Skill

Un skill es una carpeta que contiene un archivo SKILL.md. Eso es todo. Sin compilacion, sin herramientas especiales.

.opencode/
  skills/
    mi-skill/
      SKILL.md          <- Archivo principal (obligatorio)
      referencia.md     <- Archivos auxiliares (opcional)
      script.sh         <- Scripts (opcional)

El archivo SKILL.md

Tiene dos partes: un frontmatter YAML con metadatos y el contenido con instrucciones para el agente.

---
name: test-generator
description: Genera tests unitarios para archivos TypeScript. Usar cuando el usuario pida crear tests o cobertura de codigo.
---

# Generador de Tests

## Pasos a seguir
1. Lee el archivo fuente indicado
2. Identifica las funciones exportadas
3. Crea un archivo de test usando vitest
4. Incluye casos edge y manejo de errores

## Ejemplo de output
Usa `describe()` y `it()` con nombres descriptivos.
Tip: El campo description es lo mas importante. Es la unica cosa que el agente lee al inicio para decidir si cargar el skill o no.

Campos del Frontmatter

CampoRequeridoDescripcion
nameSi1-64 caracteres, minusculas con guiones. Debe coincidir con el nombre de la carpeta.
descriptionSi1-1024 caracteres. Describe que hace Y cuando usarlo. Es el "gatillo" de activacion.
licenseNoLicencia del skill (ej: MIT).
compatibilityNoRequisitos del entorno.
metadataNoPares clave-valor arbitrarios (autor, version, categoria).

Donde colocar los Skills

OpenCode busca skills en varias ubicaciones, de menor a mayor alcance:

UbicacionAlcanceUso recomendado
.opencode/skills/ProyectoSkills especificos del proyecto, compartidos via Git
.claude/skills/ProyectoCompatibilidad con Claude Code
.agents/skills/ProyectoDirectorio alternativo
~/.config/opencode/skills/GlobalSkills personales usados en todos los proyectos
Tip: Los skills en .opencode/skills/ se comparten con tu equipo via Git. Los globales en ~/.config/opencode/skills/ son solo para ti.

Como crear tu primer Skill

  1. Crea la carpeta
    mkdir -p .opencode/skills/test-generator
  2. Crea el archivo SKILL.md
    Escribe el frontmatter con name y description, luego las instrucciones para el agente.
  3. Reinicia OpenCode
    Los skills se cargan al inicio. Usa /new o reinicia la sesion.
  4. Prueba!
    Simplemente escribe algo como "genera tests para src/auth.ts" y el agente cargara el skill automaticamente.

Ejemplo Practico: Skill de Revision de Seguridad

---
name: security-review
description: Revisa codigo en busca de vulnerabilidades de seguridad OWASP. Usar cuando el usuario pida revisar seguridad, auditar modulo o detectar fallas.
---

# Revision de Seguridad

## Protocolo
1. Identifica puntos de entrada de datos del usuario
2. Busca inyecciones SQL, XSS, CSRF
3. Revisa autenticacion y autorizacion
4. Verifica que secrets no esten hardcodeados
5. Emite reporte con severidad (critico/alto/medio/bajo)

## Reglas
- Nunca omitas archivos por "parecer seguros"
- Prioriza datos criticos y endpoints publicos
- Incluye el archivo y linea exacta de cada hallazgo

Como funciona por dentro

Nivel 1: Metadata (siempre en memoria)

Al iniciar, OpenCode carga solo name y description de cada skill. ~100 tokens por skill. Con 50 skills, solo 5,000 tokens.

Nivel 2: Instrucciones (bajo demanda)

Cuando el agente decide que un skill es relevante, lee el contenido completo del SKILL.md. Recomendado: menos de 500 lineas.

Nivel 3: Recursos (bajo demanda)

Si el SKILL.md referencia archivos adicionales, el agente los carga solo cuando los necesita.

Instalar Skills existentes

Usa el CLI npx skills para instalar skills de la comunidad:

# Cloudflare Workers, AI, KV, R2, D1
npx skills add https://github.com/cloudflare/skills

# AI models via Replicate
npx skills add replicate/skills

# UI/UX design patterns de Anthropic
npx skills add https://github.com/anthropics/skills --skill frontend-design

# Creador de skills (para hacer tus propios skills)
npx skills add https://github.com/anthropics/skills --skill skill-creator
Despues de instalar, reinicia OpenCode para que cargue los nuevos skills.

Permisos de Skills

Puedes controlar que skills puede usar cada agente en opencode.json:

{
  "permission": {
    "skill": {
      "*": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
ValorComportamiento
allowEl skill se carga automaticamente sin preguntar
denyEl skill nunca se carga
askEl agente pregunta antes de cargar el skill

Buenas Practicas

Una responsabilidad

Cada skill debe hacer una cosa bien. No mezcles generacion de tests con revision de codigo.

Descripcion especifica

Incluye palabras clave que el agente pueda detectar. "Genera tests para TypeScript" es mejor que "ayuda con codigo".

Instrucciones concretas

Mientras mas detallado el SKILL.md, mejores resultados. Incluye ejemplos concretos.

Versiona tus skills

Usa el campo version en frontmatter y commitea los skills al repositorio.

Alcance correcto

Skills de workflow van globales (~/.config/). Skills de arquitectura van en el proyecto (.opencode/).

Prueba incrementalmente

Empieza simple, agrega complejidad gradualmente. No intentes hacer todo de golpe.

Commands vs Skills vs Agents

ExtensionSe activa conUbicacion
Custom CommandsUsuario escribe /comando.opencode/commands/*.md
Agent SkillsModelo detecta por descripcion.opencode/skills/
Custom AgentsTab o @agente.opencode/agents/*.md
Regla simple: Commands son invocados por el usuario. Skills son descubiertos por el modelo. Agents son entidades independientes con su propia personalidad.

Sinopsis

En una pequena aldea austríaca de 1590, atrapada en la Edad Media y sumida en la supersticion, aparece un extrano joven llamado Satanas, sobrino del Satanas biblico. Acompanado de tres sobrinos mas jovenes, este ser sobrenatural interactua con los aldeanos y en particular con cuatro jovenes: Theodor, Nikolaus, Seppi y mi narrador.

Satanas posee poderes ilimitados: puede crear vida, leer pensamientos, hacerse invisible y manipular la realidad. Sin embargo, lejos de ser malvado, se muestra curioso, burlon y profundamente escéptico de la naturaleza humana y de la religion organizada.

Temas Principales

La Naturaleza de la Realidad

El tema central: toda la existencia humana es un sueño grotesco de un solo ser pensante. No existe Dios, ni universo, ni infierno — solo un pensamiento nomada vagando por el vacio de la eternidad.

Critica a la Religion

Satanas desnuda las contradicciones de Dios: habla de misericordia pero inventa el infierno, impone normas morales que el mismo no cumple, y responsabiliza al hombre de sus propios actos.

El Poder de la Imaginacion

Satanas crea figuras de arcilla dotadas de vida y conciencia, demostrando que la creacion es un acto de voluntad pura. Los personajes creados existen solo mientras el pensamiento los sostiene.

Lacondicion Humana

Los aldeanos representan la ignorancia, el fanatismo y la crueldad medieval. El padre Adolfo y el padre Pedro encarnan la hipocresia religiosa que Satanas desafia constantemente.

Estructura Narrativa

La novela esta narrada en primera persona por un joven aldeano que se convierte en el companero inseparable de Satanas. A lo largo de 11 capítulos, asistimos a una serie de episodios que van desde lo comico (los intentos del padre Adolfo por exorcizar a Satanas) hasta lo filosofico (la creacion y destruccion de civilizaciones enteras).

El Final Filosofico

En el climax de la novela, Satanas revela al narrador la verdad ultima: "No existe Dios, ni el universo, ni la raza humana, ni la vida terrenal, ni el cielo, ni el infierno. Todo es un sueño, un sueño grotesco y disparatado. Nada existe sino tu. Y tu no eres sino un pensamiento, un pensamiento nomada, inutil, sin hogar propio, que vagabundea desamparado por el vacio de las eternidades."

Satanas desaparece, dejando al narrador anonadado con la certeza de que todo cuanto le habia dicho era verdad.

Contexto del Autor

Nota: Mark Twain escribio esta obra en sus anos mas oscuros, despues de perder a su esposa y sus dos hijos. El humorismo caracteristic de La rana saltarina cede aqui a la ironia y al escepticismo mas profundo. Fue publicada postumamente porque Twain consideraba que el publico no estaria preparado para su filosofia tan radical durante su vida.

Relevancia Actual

El Forastero Misterioso sigue siendo una de las obras filosoficas mas provocadoras de la literatura estadounidense. Su cuestionamiento de la realidad, la religion y la existencia humana anticipa ideas que luego desarrollarian existencialistas como Sartre y Camus. Es una lectura indispensable para comprender la evolucion del pensamiento de Twain y la critique social en la literatura.

Las Aventuras de Pinocho — Carlo Collodi

Ficha Tecnica

AutorCarlo Collodi (Carlo Lorenzini)
Titulo originalLe avventure di Pinocchio
Primera edicion1883, Florencia, Italia
GeneroLiteratura infantil, Fantasia, Aventura
Capitulos36 capitulos
TraduccionFredy Ordonez (Edicion IDARTES, Bogota, 2012)

Sinopsis

El viejo carpintero Geppetto recibe un pedazo de madera que se niega a ser tallado porque puede hablar y llorar. Geppetto decide fabricar una marioneta a la que llama Pinocho. Desde el primer momento, la marioneta cobra vida y se escapa, iniciando una serie de aventuras y desventuras que lo llevan por caminos inesperados.

Pinocho es un niño travieso, mentiroso e irresponsable que constantemente desobedece a Geppetto y a la Niña del Pelo Turquesa, su hada madrina. A lo largo de su camino se encuentra con personajes memorables: el Gato y el Zorro (dos estafadores), el Grillo Parlante (su conciencia), el Comisario, y muchos otros.

Las aventuras de Pinocho incluyen ser engañado, convertirse en un burro, ser devorado por un tiburon (o ballena, segun la edicion), y finalmente aprender que la honestidad, la obediencia y el amor al estudio lo transforman. Al final de su historia, Pinocho se convierte en un nino de verdad.

Temas Principales

La Honestidad

La mentira es el pecado principal de Pinocho: cada vez que miente, le crece la nariz. La obra ensena que la honestidad es la virtud fundamental para ser una buena persona.

La Educacion y el Estudio

Pinocho prefiere jugar y divertirse en vez de ir a la escuela. La obra muestra las consecuencias de ignorar el aprendizaje y la disciplina.

El Amor Familiar

La relacion Pinocho-Geppetto es el corazon emocional de la historia. El sacrificio del padre y la gratitud del hijo son los valores centrales.

La Transformacion Moral

Pinocho evoluciona de una marioneta desobediente a un nino responsable. Su transformacion fisica (de madera a carne) simboliza su crecimiento interior.

Personajes Principales

Pinocho

Marioneta de madera que cobra vida. Travieso, mentiroso pero con buen corazon. Su deseo mas grande es convertirse en un nino real.

Geppetto

Viejo carpintero pobre y amoroso que crea a Pinocho. Representa el amor paternal incondicional.

Niña del Pelo Turquesa

Hada madrina de Pinocho. Aparece y desaparece misteriosamente, guiando a Pinocho en sus momentos mas criticos.

Grillo Parlante

La conciencia de Pinocho. Aparece una sola vez pero su consejo se cumple: los ninos desobedientes se arrepienten.

Estructura Narrativa

La novela esta organizada en 36 capitulos que siguen la estructura clíasica del cuento de hadas: el héroe (Pinocho) emprende un viaje, enfrenta pruebas y tentaciones, fracasa repetidamente, y finalmente se redime a traves del sacrificio y la bondad. Cada aventura es una leccion moral.

Pasaje Destacado

"Cuando los ninos malos se vuelven buenos, tienen la virtud de adoptar un aspecto completamente nuevo e irradiar alegria a su familia."
— Geppetto, al ver a Pinocho transformado en nino real

Contexto del Autor

Nota: Carlo Collodi (1826-1890) fue un escritor y periodista italiano. Creo a Pinocho como una historia por entregas en una revista infantil. La historia original era mas oscura: Pinocho era un personaje egoista y desobediente que sufría castigos extremos. La version conocida hoy fue suavizada para el publico infantil. Collodi no imagino que su creacion se convertiria en una de las historias mas traducidas y adaptadas de la literatura universal.

Relevancia Actual

Las Aventuras de Pinocho es una de las obras mas traducidas del mundo (despues de la Biblia y el Coran). Ha sido adaptada en mas de 250 peliculas, series y obras de teatro, incluyendo la famosa version de Disney (1940) y el ganador del Oscar Pinocchio de Guillermo del Toro (2022). Sus temas sobre la honestidad, la educacion y la transformacion personal siguen siendo universales y relevantes para todas las generaciones.
